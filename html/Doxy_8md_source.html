49 ### How does automatic starting row detection work?
50 See "How does automatic delimiter detection work?"
51 
52 ### How does automatic delimiter detection work?
53 First, the CSV reader attempts to parse the first 100 lines of a CSV file as if the delimiter were a pipe, tab, comma, etc.
54 Out of all the possible delimiter choices, the delimiter which produces the highest number of `rows * columns` (where all rows
55 are of a consistent length) is chosen as the winner.
56 
57 However, if the CSV file has leading comments, or has less than 100 lines, a second heuristic will be used. The CSV reader again
58 parses the first 100 lines using each candidate delimiter, but tallies up the length of each row parsed. Then, the delimiter with
59 the largest most common row length `n` is chosen as the winner, and the line number where the first row of length `n` occurs
60 is chosen as the starting row.
61 
62 Because you can subclass csv::CSVReader, you can implement your own guessing hueristic. csv::internals::CSVGuesser may be used as a helpful guide in doing so.
63 
64 ### Is the CSV parser thread-safe?
65 The csv::CSVReader iterators are intended to be used from one thread at a time. However, csv::CSVRow and csv::CSVField objects should be
66 thread-safe (since they mainly involve reading data). If you want to perform computations on multiple columns in parallel,
67 you may want to avoid using the iterators and
68 use csv::CSVReader::read_row() to manually chunk your data. csv::CSVStat provides an example of how parallel computations
69 may be performed. (Specifically, look at csv::CSVStat::calc() and csv::CSVStat::calc_worker() in csv_stat.cpp).
